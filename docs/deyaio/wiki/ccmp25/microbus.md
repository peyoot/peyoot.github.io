# 开发套件上的MikroBus接口
MikroBus​ 是一种由 MikroElektronika​ 公司发明的、标准化的板对板连接器接口规范。它的核心目标是解决嵌入式开发中的 “碎片化”​ 问题。
在传统开发中，为不同的传感器、执行器、通信模块（如陀螺仪、温湿度传感器、蓝牙模块等）设计电路、焊接引脚、编写底层驱动非常耗时。MikroBus 通过定义一个物理、电气和逻辑上的统一标准，让各种功能模块可以像“积木”一样，通过简单的插接方式，快速连接到主控板上，极大地加速了原型开发和概念验证。

1. 物理尺寸：所有 MikroBus 插座和插槽（模块）遵循统一的尺寸和引脚布局。
相邻引脚中心间距2.54，每个焊盘网络1.27mm，焊盘本身0.6~0.8mm即可。 插座对应的插针直径0.64mm。

2. 引脚定义：一个标准的 MikroBus 插座包含 16 个引脚，这些引脚预先定义了明确的功能。
主要分为以下几类：
* 电源：3.3V， 5V， GND
* 数字接口：RST（复位）， CS（片选）， SCK， MISO， MOSI（SPI接口）， PWM
* 模拟接口：AN（模拟输入）， INT（中断）
* 通信总线：RX， TX（UART）， SCL， SDA（I2C）

MikroElektronika 推出了超过 1000 款​ 官方的“Click board™”模块（即符合 MikroBus 标准的子板），涵盖了传感器、执行器、通信、显示、存储等几乎所有领域。开发者无需关心模块的底层硬件连接，只需将其“Click”到主板上即可。
